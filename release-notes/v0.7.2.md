## What's New

- New sessions start reliably again, including untitled drafts that previously failed with a 400 error.
- Custom `neocoded` paths now validate immediately in Settings so broken binaries are caught before restart.
- The daemon path picker is cleaner and less cramped after removing the extra clear action.

## Improvements

- NeoCode now tells you to restart after a valid daemon path change so it is clear when the new binary will take effect.
- Daemon path validation checks both executability and exact version compatibility with NeoCode 0.7.2.

## What's New

- NeoCode now runs through a bundled `neocoded` daemon, which centralizes runtime, event streaming, Git, dashboard, and workspace orchestration behind a single local backend.
- Added managed daemon installs with version matching, architecture-aware downloads, checksum verification, and a retry flow so the app can recover when the daemon is missing or out of date.
- Improved startup and session handling with lighter dashboard caching, transcript-on-demand loading, better session stats refresh behavior, and restored rich trace/message envelopes.

## Improvements

- Reworked the release pipeline so app and daemon artifacts ship together, including macOS arm64 and amd64 daemon archives plus checksums.
- Reduced startup memory usage by removing eager transcript hydration from cached project state.
- Cleaned up transport and runtime naming to reflect the daemon-backed architecture more accurately.
